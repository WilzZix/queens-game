# Theme (Light/Dark) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a light-default / dark theme toggle to the Queens game, persisted in `localStorage`, with no build step and no flash of wrong theme on load.

**Architecture:** All colors move into CSS custom properties on `:root`; a `:root[data-theme="dark"]` block overrides them. A tiny inline boot script in `<head>` applies the stored theme before first paint (prevents FOUC). A `public/theme.js` module holds pure, unit-tested logic (`resolveTheme`, `nextTheme`) plus thin DOM helpers (`applyTheme`, `toggleTheme`, `setupThemeToggle`). `game.js` calls `setupThemeToggle` once on load.

**Tech Stack:** Vanilla HTML/CSS/JS, browser-native ES modules, `node:test` for unit tests, Firebase Hosting (`public/`).

**Spec:** `docs/superpowers/specs/2026-05-31-queens-cloud-features-design.md` (sub-project 1).

- [ ] **Step 3: Write minimal implementation**

Create `public/theme.js`:

```js
const STORAGE_KEY = "queens.theme";

export function resolveTheme(stored) {
  return stored === "dark" ? "dark" : "light";
}

export function nextTheme(current) {
  return current === "dark" ? "light" : "dark";
}

export function applyTheme(theme) {
  document.documentElement.dataset.theme = theme;
}

export function toggleTheme() {
  const current = document.documentElement.dataset.theme || "light";
  const next = nextTheme(current);
  applyTheme(next);
  try {
    localStorage.setItem(STORAGE_KEY, next);
  } catch {
    /* storage unavailable (private mode) — toggle still works for this session */
  }
  return next;
}

export function setupThemeToggle(btn) {
  if (!btn) return;
  const sync = () => {
    btn.textContent = document.documentElement.dataset.theme === "dark" ? "☀️" : "🌙";
  };
  sync();
  btn.addEventListener("click", () => {
    toggleTheme();
    sync();
  });
}
```

## Notes for the implementer

- Do NOT add system-preference (`prefers-color-scheme`) following — light is the deliberate
  default per spec. A stored choice is the only override.
- Region cell colors come from `PALETTE` in `game.js` and are intentionally left un-themed;
  do not move them into CSS variables.
- The inline boot script duplicates the light/dark decision on purpose — it must run before
  the stylesheet paints, which a deferred ES module cannot guarantee.
